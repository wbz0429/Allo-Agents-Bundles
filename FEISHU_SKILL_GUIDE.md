# 飞书 Skill 开发指南（接手必读）

> 读完这一份,你就能给自己的 Agent Bundle 接上飞书 —— **不用改 Allo,不用翻飞书文档,不用碰飞书 API**。
> 适合直接拿去 vibe coding:照着 §2 的两步 + §4 的示例,让你的 bundle 把内容发成飞书卡片。

---

## 0. 先记住一句话

> **平台管「通道和信封」,你的 bundle 管「信的内容」。**

- **平台(Allo 底层)** 已经替你打通了飞书:连接、鉴权、收发、会话路由、**把文本渲染成飞书卡片**。这些你都不用做。
- **你(bundle/skill)** 只负责:**说清要发什么内容、卡片有哪些字段**。

判断某件事归不归你:**「换一个飞书 agent 要不要重写?」要 → 归你;不要 → 平台已做好。**

---

## 1. 为什么你不用调飞书 API（重要）

我们采用「厚平台 / 薄 skill」:**只有平台持有飞书凭据、只有平台调飞书 API。** 你的 skill **不碰** `app_id/app_secret`、**不拼**飞书卡片 JSON、**不发** HTTP 请求。你只产出"内容",平台负责发。

好处:你的包里**永远不会出现密钥**,也不必学飞书那套卡片 schema。

---

## 2. 你只需要做两件事

### ① 在 `capabilities.yaml` 里声明标题和字段

```yaml
display_name: 我的助手            # 飞书卡片标题就取这里
policies:
  feishu_card_labels: [状态, 范围, 指标, 诊断, 建议, 缺口]   # 卡片有哪些字段，按顺序展示
```
- **`feishu_card_labels` 第一个永远是"状态"语义位**(决定卡头颜色),其余即展示顺序。
- 不写就用平台默认的监控模板字段;字段名你随便改,平台都能渲染。

### ② 让你的 agent 输出 `标签：值` 文本

agent 正常回话时,按你声明的标签输出文本即可(中文 `：` 或英文 `:` 都行):

```
状态：正常
范围：最近 24h
指标：qps=1280 p99=180 ms
诊断：无异常
建议：保持观察
```

平台收到后**自动渲染成飞书卡片**。就这两步,飞书就通了。

---

## 3. 平台的渲染规则（知道它怎么变成卡片）

- **至少命中 2 个标签**才当结构化卡片;否则按普通 markdown 卡片发(直接出 markdown 也行,表格会自动降级成 `值 · 值` 行)。
- **第一个标签 = 状态 → 卡头颜色**:绿 `正常/健康/稳定`、橙 `关注/预警/警告`、红 `异常/故障/严重`、灰 `数据不足/无数据/未知`、其余蓝。
- **`指标`** 渲染成要点列表(支持 `key=value unit; key2=value2`)。
- **`缺口`** 渲染成 note 备注。
- 其余标签各自成一段。不认识的标签**优雅降级**成普通段落,不会报错。
- 卡片标题 = `display_name`;底部会自动附"回复本条可继续会话"的提示,你不用管。

---

## 4. 一个完整最小示例

**capabilities.yaml**(节选)
```yaml
display_name: 库存助手
policies:
  feishu_card_labels: [状态, 范围, 指标, 建议]
```

**agent 输出**
```
状态：关注
范围：华东仓 / 最近 1h
指标：缺货SKU=12 周转天数=43
建议：优先补货 3 个高频 SKU
```

**用户在飞书看到**:一张**橙色**卡头("关注")的卡片,标题"库存助手",依次是状态+范围概览、指标要点列表、建议段落。

你写的全部就是上面两块 —— 没有一行飞书 API 代码。

---

## 5. 红线（绝不能做）

- **包里禁止出现明文密钥**:任何凭据只能写 `$ENV` 引用(如 `$FEISHU_APP_SECRET`),由平台注入。
- **不要在 skill 里自己调飞书 OpenAPI / 发 HTTP** 去推消息 —— 产出内容交给平台。
- **不要硬编码卡片颜色 / schema / 飞书接口**:这些是平台的事,会随平台升级。

### 一条已知例外（别照抄）
"**主动推送**(没人 @ 你时主动把卡片推到群)"和"**查群号**"平台目前还没提供能力,`xingyuan-monitor` 的 `feishu-webhook-report` skill 临时自己调了飞书 API 顶上。它仍守住了"密钥不落明文"的红线,但这是**过渡写法**,平台后续会把这能力收编。**你写新 skill 时不要照抄它自调 API 的做法。**

---

## 6. yes / no 速查

| 我想做的 | 归谁 | 怎么做 |
|---|---|---|
| 改卡片标题 | 你 | 设 `display_name` |
| 卡片加一个"风险等级"字段 | 你 | `feishu_card_labels` 加 `风险等级`,正文输出 `风险等级：高` |
| 改卡头配色规则 | 平台 | 不在 bundle 改 |
| 只在被 @ 时才回 | 你(配参数) | `channels.feishu.require_mention: true` |
| 发带按钮的交互卡 | 平台待办 | 超出当前能力 |
| skill 里写密钥 / 调飞书 API | ❌ 禁止 | 撞红线(主动推送例外见 §5) |
